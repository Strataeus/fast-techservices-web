---

## 0) Mission
Créer un site **premium, high-tech, industriel**, qui convertit vers **FAST Remote** (offre cœur) et génère des demandes onsite qualifiées, tout en respectant la doctrine FAST :
- **TERRAIN → PREUVE → VERDICT**
- **PAS DE PREUVE = PAS FAIT**
- **UNKNOWN obligatoire** si donnée critique manquante → **SAFE-BY-DEFAULT**
- **STOP & CALL** sur risque/doute
- **Responsabilité finale HUMAINE** (IA jamais décideur)
- **FASTCore non exposé** au public ; site → intake gateway → FASTCore

Ce site est une **machine à leads** et un **outil de crédibilité** (preuves, méthode, livrables).

---

## 1) Objectifs mesurables (KPIs)
### 1.1 Conversion & business
- KPI1 : Taux de conversion formulaire **FAST Remote** ≥ 2% (à ajuster selon trafic).
- KPI2 : 100% des soumissions formulaire reçoivent un **accusé immédiat** + sont **loggées** côté serveur.
- KPI3 : 0 lead perdu (pas de page “Chargement…” / widget cassé / spam non filtré).
- KPI4 : 3 études de cas réelles minimum publiées (sinon affichage “propre” sans fake preuves).

### 1.2 Qualité & performance
- Lighthouse : Perf/SEO/Best Practices/Accessibility ≥ 90 (cible).
- Temps de chargement (LCP) : ≤ 2.5s sur mobile (cible).
- Aucune dépendance inutile lourde (jQuery/wow etc. éliminées si non indispensables).

### 1.3 Cohérence & conformité
- Un seul **SLA** affiché partout, source unique (config).
- Zéro placeholder visible en prod.
- Mentions légales + confidentialité + cookies conformes FR.

## 2) Positionnement (brand)
### 2.1 Définition
**FAST Tech Services** : interventions électromécaniques premium sur équipements techniques de garages automobiles :  
- ponts élévateurs  
- compresseurs & réseaux d’air comprimé  
- cabines de peinture & ventilation/traitement d’air  
- stations de lavage  
+ retrofit, intégration, mise en service, fiabilisation.

### 2.2 ADN
- Terrain-first, environnements critiques (sécurité/qualité/disponibilité).
- Méthode structurée + preuves + tests de sortie.
- FAST Remote = diagnostic et assistance visio, orienté décision rapide.

### 2.3 Promesse
- “Remise en exploitation propre” + “diagnostic guidé” + “dossier clair”.
- Pas de promesse marketing non prouvable.

---

## 3) SLA unique (obligatoire et identique partout)
**SLA FAST Remote / Contact**
- Accusé automatique : immédiat
- Réponse humaine : **≤ 4h ouvrées**
- Créneau visio : **≤ 24h ouvrées** (J+1 ouvré)
- Verdict / plan d’action : **≤ 2h** après session (si preuves/mesures suffisantes), sinon **UNKNOWN + plan**

> Toute autre valeur SLA est INTERDITE. Source : `content/config.json` (à créer).

---

## 4) UX/Copywriting (règles non négociables)
- FR uniquement, vouvoiement uniforme.
- Ton : technique, premium, clair, sans jargon inutile.
- Toujours expliquer ce que le client obtient : livrables, verdict, réserves.
- Supprimer toute “statistique” non prouvable (avis, notes, % succès, etc.).
- CTA unique principal : **Démarrer FAST Remote**.
- CTA secondaire : **Demander une intervention onsite**.

## 5) Architecture site (Next.js App Router)
### 5.1 Structure recommandée
- `apps/web/app/(site)/` : pages publiques
- `apps/web/app/api/` : routes serverless (formulaires)
- `apps/web/components/` : UI + sections
- `apps/web/content/` : données (JSON/MDX)
- `apps/web/public/` : assets (images, icônes, vidéo)

### 5.2 Rendering
- Pages marketing : SSG/SSR HTML-first (priorité SSG).
- JS minimal : animations sobres, pas de libs lourdes.

### 5.3 Design system
- Dark industriel + accents cyan/bleu énergie.
- Typo moderne, hiérarchie claire.
- Layout : spacing généreux, grid propre, cohérence.

## 8) Formulaires & Intake (intégration FASTCore/FAST IA)
### 8.1 Règles
- Le site ne contacte JAMAIS FASTCore directement.
- Le site appelle un endpoint intake configurable `INTAKE_URL`.
- Toute soumission doit être loggée côté serveur (request_id).

### 8.2 Endpoints web
- `POST /api/leads` : réception lead + validation + anti-spam + forward intake.

### 8.3 Payload lead (schéma)
- type: `fast_remote` | `onsite` | `contact`
- nom, société, email, téléphone
- ville, code postal
- équipement (catégorie + marque/modèle/serial si dispo)
- symptôme (texte)
- urgence (niveau)
- disponibilité
- consentement RGPD (bool)
- meta: referer, utm, user_agent (hash IP optionnel)

### 8.4 Réponse utilisateur
- Succès : message + email accusé (si SMTP)
- Échec : message clair, pas de crash
